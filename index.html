04 / skill tree

What unlocks what

Sequential dependencies. TF2 and Gazebo are hard blockers.

CORE PATH
Linux CLI
├── ROS2 workspace (colcon, ament)
│   ├── Topics + Pub/Sub
│   │   ├── Services (.srv)  ← you are here
│   │   ├── Actions (.action)
│   │   └── Parameters (YAML)
│   └── TF2 ← bottleneck #1
│       └── URDF + RViz2
│           └── Gazebo ← bottleneck #2
│               └── Nav2 / MoveIt2
│                   └── SLAM → Jobs
Bottleneck #1 — TF2
Master: parent_frame, child_frame, lookupTransform, time tolerances. Everything in Nav2 and SLAM depends on this.
Bottleneck #2 — Gazebo wiring
ros_gz_bridge topic remapping is non-obvious. Spend extra time here before Nav2.
HIRE-BOOSTERS
C++ nodes (rclcpp) · SLAM Toolbox deep dive · OpenCV + image_transport · Docker + GitHub CI · ros2_control
